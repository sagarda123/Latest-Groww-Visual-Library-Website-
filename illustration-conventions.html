Illustration naming conventions

Every illustration in the library follows one predictable pattern so designers and engineers can find, reference, and validate assets without guesswork. The library flags any name that breaks these rules.

1 · The pattern

Lowercase snake_case, three parts, in this order:

mds_il Prefix Required
hero | spot_hero | illustrated_icon Illustration type Required
risk_fno Illustration name Required

Validation regex: ^mds_il_(hero|spot_hero|illustrated_icon)_[a-z0-9_]+$

2 · Illustration types

mds_il_hero_*

Full-bleed hero artwork for empty states, errors, and feature intros. The biggest, most detailed pieces.

50 assets in the library
mds_il_spot_hero_*

Mid-size spot artwork used inline in cards, sheets, and banners — lighter than a hero, richer than an icon.

55 assets in the library
mds_il_illustrated_icon_*

Small illustrated icons (native 64 px) for lists, categories, and product entry points.

43 assets in the library

4 · Do & don't

RuleDoDon't
Lowercase snake_case onlymds_il_hero_stocks_sipmds_il_hero_Stocks_SIP
Type token comes from the three fixed valuesmds_il_spot_hero_calendarmds_il_banner_calendar
Name describes the content, not the screen it's used onmds_il_hero_empty_watchlistmds_il_hero_homepage_v2_final
No brand or theme tokens — light/dark are file variants, not namesmds_il_hero_win_trophymds_il_hero_win_trophy_dark_groww
No spaces, hyphens, or special charactersmds_il_illustrated_icon_mf_nfomds-il illustrated (icon) NFO!

Rules are enforced in lib/rive/namingRules.js — the library shows a red ⛔ badge beside any name that fails, and the upload dialog validates before anything is added.
