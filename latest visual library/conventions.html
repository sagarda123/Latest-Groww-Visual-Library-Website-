01 — Files

File naming

Every .riv file name is an address — snake_case, broad to narrow. Sequence numbers use an s-prefix, versions use a v-prefix — making both unambiguous.

mds_riveMDS prefixrequired
[type]Animation typerequired
[_product?]Productoptional
[_feature?]Featureoptional
[_s{n}?]Sequenceoptional
[_v{n}?]Versionoptional
.rivExtensionrequired
Animation types
ftuxgtm_storyspotloaderorder_statusiconspot_heroherolanding_page_herologo
Products
stocksfnochartsmfloans
Sub-products / features
mtfintradaytechnicals
Extensible — needs MDS approval

Examples

mds_rive_loader_stocks_mtf.rivLoader for MTF in Stocks
mds_rive_gtm_story_stocks_s2.rivSecond file in a GTM story sequence — note s prefix
mds_rive_loader_stocks_mtf_v2.rivRedesigned loader — v1 still live, must coexist
mds_rive_ftux_mf.rivFirst-time UX for Mutual Funds
mds_rive_order_status_fno.rivOrder status for F&O
mds_rive_logo.rivGeneric logo — no product context
Sequence vs version: _s2 = file 2 of a multi-file project (all files needed together). _v2 = redesigned replacement, only used while the old version is still alive in production.
02 — Prefix system

In-file naming: two prefixes

Everything named inside a Rive file follows one rule. The prefix casing tells you whether the thing is a container or a descriptor — no lookup required.

PascalCase · RiveX
Things — containers and entities
RiveXAbLoaderRiveXCompButtonRiveXSmLoadingStatusRiveXVmButtonRiveXEnumThemeMode
camelCase · riveX
Descriptors — properties, timelines, events, inputs
riveXBooleanIsLoadingriveXEventOnCompleteriveXHitAreaCardriveXTimelineOnPressriveXImageAvatar
RiveX for containers — canvases, data contracts, behaviour graphs. riveX for descriptors — properties, events, timelines.
03 — Canvases

Artboards & Components

Artboard naming uses the two-char Ab abbreviation (matching Vm and Sm). RiveXComp names artboards exported as reusable components via the Rive editor.

Artboards — RiveXAb

RiveXAbFixed prefixrequired
[AnimationType]Animation typerequired
[IconOrIllustration?]Icon / illustrationoptional
RiveXAbFtuxOnly artboard in an FTUX file
RiveXAbOrderStatusSuccessSuccess artboard in an order-status file
RiveXAbIconWalletWallet artboard in a multi-icon file

Components — RiveXComp

Export as Component turns an artboard into a reusable, instance-aware building block. Each component instance gets its own View Model Instance.
RiveXCompFixed prefixrequired
[Name]Namerequired
RiveXCompButtonReusable button with Idle/Hover/Pressed/Disabled states
RiveXCompToastNotification toast component
RiveXCompProgressBarData-bound progress indicator
RiveXCompAvatarUser avatar with online/offline indicator
RiveXCompBadgeNumeric badge for counts
RiveXCompSparklineInline data-bound chart row
Ab = artboard, Comp = component
Ab is the canvas. Comp is the exported, instance-aware building block. A Comp always corresponds to an Ab underneath.
PascalCase, no underscores
Both RiveXAb and RiveXComp use PascalCase concatenation throughout — no separators between segments.
Mark instances as Exported
For a View Model Instance to be visible to runtime developers, it must be marked Exported in the Rive editor.

Instances

Pre-configured presets of View Models, State Machines, and Artboards. The artboard abbreviation is Ab.

riveXInstanceFixed prefixrequired
[Vm|Sm|Ab]Source typerequired
[SourceName]Source namerequired
[Variant]Variantrequired
riveXInstanceVmButtonPrimaryParent RiveXVmButton · Variant Primary
riveXInstanceVmButtonSecondaryParent RiveXVmButton · Variant Secondary

Exported instances. Mark instances that developers should consume as Exported in the Rive Data panel. Unexported instances are internal test fixtures — the runtime cannot reference them by name.
06 — Inputs

View model properties

Three segments, always in this order. Includes Image, List, and ViewModel types, and the boolean directionality rule.

riveXPrefixrequired
[Type]Property typerequired
[Action]Action / staterequired
riveXTriggerOnPressTrigger — Fire-and-forget — goes from runtime INTO Rive
riveXBooleanIsLoadingBoolean — Use Is or Has prefix; true = active state
riveXEnumCompletionStatusEnum — Named switch; use the RiveXEnum type as the type slot
riveXNumberScrollProgressNumber — Any numeric value — progress, speed, count
riveXStringLabelString — Text content injected at runtime
riveXColorAccentColor — Dynamic color value — theme, branding
riveXImageAvatarImage — Runtime-swappable image slot
riveXListFeedItemsList — Array of nested VM instances — for repeating rows
riveXViewModelConfigViewModel — Nested View Model — for compound data structures
Breaking change — Boolean directionality rule. Boolean VM property names must be phrased so that true = the active / on state. Use an Is or Has prefix.
✗riveXBooleanThemeDark — Ambiguous — is true dark or false dark?
✓riveXBooleanIsDark — Clear — true = dark mode is on
✗riveXBooleanLoading — No Is/Has prefix — direction unclear
✓riveXBooleanIsLoading — Clear — true = currently loading
✗riveXBooleanError — Noun, not a boolean state
✓riveXBooleanHasError — Clear — true = error is present

07 — Interaction

Events & hit areas

Events are the outbound channel — Rive fires them at the runtime. Hit areas are the invisible shapes that make interactions reliable.

Rive Events — riveXEventOn

Events fire FROM Rive → TO the runtime
Use for: animation complete, form submitted, threshold crossed. The runtime registers a listener and reacts.
riveXEventPrefix
On[Verb]Trigger
riveXEventOnCompleteAnimation or flow has finished
riveXEventOnSubmitUser confirmed an action inside Rive
riveXEventOnErrorAn error state was reached
riveXEventOnThresholdA value crossed a defined threshold
riveXEventOnLoopA looping animation restarted
riveXEventOnSkipUser skipped an intro animation
riveXTrigger[Action]
Runtime → Rive (inbound). Your app fires a trigger to tell Rive to do something.
riveXTriggerOnPress
riveXEventOn[Verb]
Rive → Runtime (outbound). Rive fires an event to tell your app something happened.
riveXEventOnComplete

Hit Areas — riveXHitArea

Invisible shapes used as listener targets
Never attach listeners to visible art. Create a transparent rectangle for the tap region, name it riveXHitArea, and attach the listener there.
riveXHitAreaPrefix
[Target]Target
riveXHitAreaButtonFull button tap surface
riveXHitAreaCardFull card tap region
riveXHitAreaFullscreenEntire artboard as a tap target
Invisible ≠ zero opacity on the fill. Set the shape's fill to transparent (or 0% opacity) but keep it in the hierarchy above the art so it receives pointer events. If you set the whole object's opacity to 0, some runtimes won't hit-test it.
08 — Motion

Timelines

Distinguishes event-driven timelines (riveXTimelineOn…) from state-driven timelines (riveXTimeline…) to make each animation's purpose scannable in the Animations panel.

State-driven · riveXTimeline[State]
Loops and states — play continuously or hold a visual state
riveXTimelineIdleriveXTimelineSuccessriveXTimelineErrorriveXTimelineThemeDarkriveXTimelineThemeLightriveXTimelineStory
Event-driven · riveXTimelineOn[Verb]
Triggered by user interaction — plays once in response to an action
riveXTimelineOnPressriveXTimelineOnHoverriveXTimelineOnReleaseriveXTimelineOnCompleteriveXTimelineOnSkip
09 — Typography

Fonts

Two families, lean weights, English names. Font files can be the single biggest line item in a .riv — subset aggressively.

Groww Sans · Primary brand typeface
Aa Regular → GrowwSansRegular
Aa Medium → GrowwSansMedium
Aa Bold → GrowwSansBold
Söhne · Display & editorial Rename on upload
Aa Söhne Kräftig → Regular → SohneRegular
Aa Söhne Halbfett → Bold → SohneBold
One file per weight
Upload each font weight separately — never bundle multiple weights into one Rive font asset.
Minimize weights
Fewer weights = smaller .riv files and faster loads. Include only weights actually rendered in the animation.
Tick Referenced + Glyphs Used
In the Rive font upload, enable both options. This embeds only the characters actually used — can cut font size by 80%.
Rename Söhne before uploading. German weight names (Kräftig, Halbfett) are unsearchable in English codebases. Rename to Regular / Bold before adding the font to the Rive file.
